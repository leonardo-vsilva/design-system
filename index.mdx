## Princípios

<CardGroup cols={2}>
  <Card icon="layer-group" title="Consistência acima de criatividade">
    Prefira componentes existentes a soluções ad-hoc. Se existe um componente para o caso, use-o.
  </Card>
  <Card icon="palette" title="Sem customizações de cor">
    Nunca adicione cores fora dos tokens definidos. O tema é neutro/acromático — preto, branco e escala de cinzas.
  </Card>
  <Card icon="moon" title="Dark mode sempre">
    Todos os componentes devem funcionar nos dois temas. O dark mode é ativado pela classe `.dark` no elemento pai.
  </Card>
  <Card icon="code" title="Semântica primeiro">
    Use o componente correto para o contexto correto. Não use `Button` para navegação — use `Link`.
  </Card>
</CardGroup>

## Cores com matiz

O tema da Linkana é acromático. Cores com matiz aparecem **apenas** em contextos funcionais:

| Cor      | Token                 | Uso                        |
| -------- | --------------------- | -------------------------- |
| Vermelho | `destructive`         | Erros, ações irreversíveis |
| Âmbar    | `warning`             | Alertas de atenção         |
| Verde    | `success`             | Confirmações, sucesso      |
| Variado  | `chart-1` a `chart-5` | Visualizações de dados     |
